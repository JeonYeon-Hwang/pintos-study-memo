## Virtual Memory: Mmap

**mmap함수에 대한 상세 이해**
mmap은 기본적으로 file_baked_page를 만들기 위한 필수 함수이다.
일반적인 방식 .. 디스크 =(read)=> 커널 램 => 유저 램 .. 으로 옮겨진다.
이 방식은 디스크의 것이 램에 그대로 복제되는 것일 뿐이며, 이후 서로는 분리됨.
이것을 다시 write을 하여 램에서 변경 내용을 디스크에 반영한다.

*file_baked_page의 mmap은 무엇이 다른가?*
디스크의 file 특정 영역과 램에 띄워진 페이지 주소는 유기적으로 연결된 것이 특징.
해시함수의 key = value 처럼 디스크 주소와 페이지 (가상) 주소는 연결이 되어 있다.

*왜 연결을 해놓았는가?*
결론은.. 디스크에 쓰기(I/O) 부하를 감소하려는 목적. 
램 상의 가상 파일에다가 계속 반영하다가, 나중에 실제 디스크에 일괄 반영하면 되니까.

*그냥 나중에 일괄 write하면 되지 않는가? .. 굳이 mmap사용 이유?*
복사 비용(Copy Overhead)에서 큰 차이.. 기존이라면, 유저 램 => 커널 램 => 디스크
경로를 매번 밟는데, 여기서 copy 비용이 막대하다.
mmap으로 커널 램에 단 하나의 영역을 만들고, 유저 영역의 페이지가 이를 바라보게 하면,
매번 copy 비용이 발생 안하며, 여러 프로세스에서 동일 커널 램 참조가 가능해진다.
